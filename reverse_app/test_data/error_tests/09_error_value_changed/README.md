# エラーテストケース: 値の変更（Sentence要素内の子要素と値自体の変更を含む）

## テスト目的
逆変換スクリプトが、以下の値の変更を検出できるかを検証します：
1. Sentence要素内の子要素（Ruby要素、Sup要素、Sub要素など）の欠落
2. 値自体の変更（テキストの変更）

## テスト内容
- **input.xml**: Sentence要素内にRuby要素、Sup要素、Sub要素を含む正常な階層構造のXML
- **expected.xml**: すべての子要素と正しい値を含む正しい変換後のXML
- **error.xml**: 以下のエラーを含む変換後のXML
  - エラー1: Ruby要素が欠落
  - エラー2: 値自体が変更（「（変更された値）」→「（別の値）」）
  - エラー3: Sup要素とSub要素が欠落
  - エラー4: 値自体が変更（「Subitem1-2の内容」→「Subitem1-2の別の内容」）

## 検証フロー
1. `input.xml`を逆変換パイプラインで処理 → `output.xml`を生成
2. `output.xml`と`expected.xml`を比較 → 一致すれば正常な変換結果
3. `output.xml`と`error.xml`を比較 → 不一致を検出できればエラー検出成功

## 検証項目
1. 正常な変換結果が得られるか（output.xml == expected.xml）
2. Sentence要素内の子要素（Ruby要素、Sup要素、Sub要素など）の欠落を検出できるか
3. 値自体の変更（テキストの変更）を検出できるか
4. 複数のエラー（子要素の欠落と値の変更）を同時に検出できるか
