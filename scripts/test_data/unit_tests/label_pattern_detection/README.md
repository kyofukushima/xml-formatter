# ラベルパターン検証テストデータ

このフォルダには、ラベルパターン検出機能の検証用テストデータが含まれています。

## テスト目的

`scripts/utils/label_utils.py`の`detect_label_pattern`関数が、以下のラベルパターンを正しく区別できるかを検証します。

## テスト対象パターン

### 数字系
1. **全角数字**（０-９）: `０`, `１`, `２` など
2. **半角数字**（0-9）: `0`, `1`, `2` など
3. **漢数字**（一二三四五六七八九十百千）: `一`, `二`, `三` など

### アルファベット系
4. **大文字アルファベット**（A-Z）: `A`, `B`, `C` など
5. **小文字アルファベット**（a-z）: `a`, `b`, `c` など
6. **全角大文字アルファベット**（Ａ-Ｚ）: `Ａ`, `Ｂ`, `Ｃ` など
7. **全角小文字アルファベット**（ａ-ｚ）: `ａ`, `ｂ`, `ｃ` など

### 括弧付き数字系
8. **括弧付き全角数字**: `（０）`, `（１）`, `（２）` など
9. **括弧付き半角数字**: `(0)`, `(1)`, `(2)` など
10. **括弧付き漢数字**: `（一）`, `（二）`, `（三）` など

### 括弧付きアルファベット系
11. **括弧付き大文字アルファベット**: `（A）`, `（B）`, `(A)`, `(B)` など
12. **括弧付き小文字アルファベット**: `（a）`, `（b）`, `(a)`, `(b)` など
13. **括弧付き全角大文字アルファベット**: `（Ａ）`, `（Ｂ）`, `(Ａ)`, `(Ｂ)` など
14. **括弧付き全角小文字アルファベット**: `（ａ）`, `（ｂ）`, `(ａ)`, `(ｂ)` など

### 二重括弧付き数字系
15. **二重括弧付き全角数字**: `（（０））`, `（（１））`, `((0))`, `((1))` など
16. **二重括弧付き半角数字**: `（（0））`, `（（1））`, `((0))`, `((1))` など
17. **二重括弧付き漢数字**: `（（一））`, `（（二））`, `((一))`, `((二))` など

### 二重括弧付きアルファベット系
18. **二重括弧付き大文字アルファベット**: `（（A））`, `（（B））`, `((A))`, `((B))` など
19. **二重括弧付き小文字アルファベット**: `（（a））`, `（（b））`, `((a))`, `((b))` など
20. **二重括弧付き全角大文字アルファベット**: `（（Ａ））`, `（（Ｂ））`, `((Ａ))`, `((Ｂ))` など
21. **二重括弧付き全角小文字アルファベット**: `（（ａ））`, `（（ｂ））`, `((ａ))`, `((ｂ))` など

### カタカナ系
22. **カタカナ**: `ア`, `イ`, `ウ` など
23. **括弧付きカタカナ**: `（ア）`, `（イ）`, `(ア)`, `(イ)` など
24. **二重括弧付きカタカナ**: `（（ア））`, `（（イ））`, `((ア))`, `((イ))` など

## テストケース構成

各テストケースフォルダには以下のファイルが含まれます：

- `input.xml`: テスト対象のラベルパターンを含むXML
- `expected.xml`: 期待される変換結果（階層構造を含む）
- `README.md`: そのテストケースの詳細説明

## ファイル分割について

階層が深くなりすぎないように、各テストケースは最大7要素程度に分割されています。各ファイルは独立した階層構造を持ち、各パターンが別の階層として扱われます。

## 使用方法

各テストケースのXMLファイルを`label_utils.py`のテストスクリプトで使用して、パターン検出が正しく動作することを確認します。
